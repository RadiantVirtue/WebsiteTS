HOW THE AGENT WORKS

REPRESENTATION

After-State Features

Rather than feeding the raw board or the current piece into the network, the agent uses an after-state formulation. For every valid placement (rotation × column), it simulates the drop and computes a 4-feature summary of the resulting board:

  • Lines cleared — normalised ÷ 4
  • Holes — empty cells with filled cells above, normalised ÷ 40
  • Bumpiness — sum of |height differences| between adjacent columns, ÷ 40
  • Stack height — normalised ÷ 200

The network never sees the current piece directly; it sees what the board would look like after each candidate action. This collapses the problem from "given state + action → value" into "given resulting state → value" — a much simpler and more stable learning target.

ARCHITECTURE

Policy Network

The policy network is a small fully-connected feedforward network that maps the 4 board features to a single scalar Q-value.

At decision time the agent enumerates all valid after-states, passes each through the network in a batched forward pass, and greedily selects the placement with the highest Q-value — no ε-greedy exploration at inference.

TRAINING

Prioritised Experience Replay

During training, transitions are stored in a priority queue rather than a uniform replay buffer. Each transition receives a priority proportional to its TD error — the difference between the network's predicted Q-value and the Bellman target.

Transitions with larger errors are sampled more frequently, because those are the ones the network is currently most wrong about. To correct the induced sampling bias, importance-sampling weights are applied when computing the gradient update, and priorities are refreshed after each learning step.

This makes training significantly more sample-efficient than vanilla DQN — especially in Tetris, where line-clearing events are rare and would otherwise be seen only infrequently with uniform sampling.
