HOW IT WORKS
REPRESENTATION

After-State Features

Rather than feeding the raw board into the network, the agent uses an after-state formulation. For every valid placement it simulates the drop and computes a 4-feature summary:

  • Lines cleared ÷ 4
  • Holes ÷ 40
  • Bumpiness ÷ 40
  • Stack height ÷ 200

This collapses "state + action → value" into "resulting state → value" - a far simpler learning target.

ARCHITECTURE

Policy Network

A small fully-connected feedforward network maps the 4 features to a single scalar Q-value.

4Input
→
64FC + ReLU
→
64FC + ReLU
→
QOutput

At inference the agent enumerates all valid after-states, runs a batched forward pass, and greedily picks the highest Q-value placement.

TRAINING

Prioritised Experience Replay

Transitions are stored in a priority queue keyed by TD error - the gap between predicted Q and the Bellman target. High-error transitions are sampled more often.

Importance-sampling weights correct the resulting bias, and priorities are refreshed after each update.

This is critical in Tetris: line-clearing events are rare, so uniform sampling would almost never encounter them.
