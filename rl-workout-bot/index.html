IMPLEMENTATION
PROBLEM

Workout programming is a sequential decision problem. Each session's exercise choices affect recovery, which constrains future sessions. Most people use a template or improvise. The question: can an RL agent make better sequential decisions than I can?

STATE REPRESENTATION

The musculoskeletal system is modelled as a bipartite graph - 15 muscle groups connected to 50 exercises. Each muscle carries a fatigue level that decays between sessions. The agent observes this fatigue state at each decision point and selects the next session's exercises.

REWARD FUNCTION

The hardest part: balancing three competing objectives - fatigue management (don't overtrain a muscle group), progressive overload (keep lifting heavier), and adherence (don't programme workouts I'll skip). It took several iterations before the reward function stopped fighting itself.

RESULTS

Three months on myself. 92% session adherence. Zero injuries. Against the preceding three months of manual programming - adjusting for diminishing returns via a fitted quadratic - the bot's policy was approximately 20% more efficient.
